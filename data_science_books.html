Data Science Books SQL Project

goal: answering questions about data science books available on Amazon
tools used: SQL (PostgreSQL with pgAdmin4)

Project details

View data cleaning file
View data exploration file

Here you can check out the original data set on kaggle.

The original dataset is a csv file with 18 fields and 946 records. I divided it into 3 different csv files; rating, price and physical_properties in order to be able to demonstrate skills working with multiple databases (JOINs) as well.

Creating tables and data cleaning

  • I created the 3 tables in PostgreSQL and copied the data into them from the csv files.
  • I cleaned the tables one by one, here are some examples of the issues and how I solved them:
    • the 'author' column had all the names in brackets and the capitalization wasn't unified
      I used the INITCAP function and took a SUBSTRING of the field
    • the 'dimensions' field contained all 3 dimensions of the books (e.g. 6 x 0.62 x 9 inches)
      in order to be able to use it for calculations later, the 3 dimensions had to be separated in different columns
      created 3 extra columns and used the SPLIT_PART function and converted them to NUMERIC datatype
    • the 'weight' field had the data sometimes in ounces, sometimes in pounds (e.g. 12.5 ounces or 1.82 pounds)
      Again I wanted to use it later, so had to unify them
      I added a column for the weight in pounds, copied only the numbers there with the SPLIT_PART function and converted the data type to NUMERIC
      I used the CASE function to differentiate between the 'ounces' and 'pounds' int the original column and either UPDATE the new column with a calculation or keep the original number

Data Exploration

The questions I was looking to answer and some notes where the queries get more complicated:
  1. What is the most expensive data science book available on Amazon?
  2. How many of the cheapest book could you buy from the price of the most expensive one?
  3. How many different authors have books available?
    note: If there is more than one author, then I decided to treat them as a pair everywhere, not looking individually.
  4. Who are the authors who have multiple books available, starting with the one that has the most?
  5. Looking for authors with the first name 'Eric' or similar.
  6. If I wanted to read all of the books, how many pages would that mean?
  7. How many of the books could NOT fit into a standard backpack, with dimensions 8*3.75*10.25 inches, one by one?
  8. What are the books with the maximum rating?
    9. Here I created a TEMP TABLE that only contains the books that have any info on rating, because we cannot use any other books in the next couple of questions.
  9. Out of the books with a rating between 4 and 5 which ones have the most 1star reviews?
  10. Categorize the books; the 5star-rated ones should be labeled 'super', 4-5 as 'good', 3-4 as 'OK' and 1-2 as 'bad'.

    11. JOINs will start from here
  11. List the ISBN numbers and the price of all the books that have at least a 200 reviews, starting with the best rating.
  12. Which book offers the "cheapest pages"?
  13. List the books with 5 star average rating, that are under 15USD and start with the longest?
    14. Here all 3 tables had to be joined.
  • I created a dashboard showing the 4 visualizations, also adding filters. You can filter for specific month of the year and specific pizza types where it makes sense.
